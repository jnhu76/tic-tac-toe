# 井字棋游戏逻辑验证测试设计文档

## 1. 文档概述

### 1.1 目的
本文档定义井字棋游戏逻辑的验证测试规范，确保游戏核心逻辑的正确性、完整性和鲁棒性。

### 1.2 范围
- 游戏棋盘状态管理
- 胜负判定逻辑
- 和棋判定逻辑
- 移动合法性验证
- 游戏状态转换
- 边界条件处理

### 1.3 参考文档
- `src/core/common.py` - 游戏基础模块
- `src/ai/train.py` - 训练模块（包含游戏逻辑）

## 3. 测试策略

### 3.1 测试类型
1. **单元测试**：针对单个函数/方法的测试
2. **集成测试**：多个组件协作的测试
3. **边界测试**：边界条件和异常情况的测试
4. **状态测试**：游戏状态转换的测试

### 3.2 测试方法
- **黑盒测试**：基于功能规格的测试
- **白盒测试**：基于代码结构的测试
- **等价类划分**：将输入划分为有效/无效等价类
- **边界值分析**：测试边界条件和临界值

## 4. 测试用例设计

### 4.1 棋盘初始化测试 (TC-INIT)

#### TC-INIT-001: 默认初始化
**目的**：验证棋盘默认初始化状态  
**前置条件**：无  
**输入**：`GameBoard()`  
**预期输出**：
- 棋盘为3x3矩阵，所有元素为None
- current_player = 0（AI先手）
- col = 3, row = 3, board_size = 9

#### TC-INIT-002: 自定义尺寸初始化
**目的**：验证棋盘支持自定义尺寸  
**前置条件**：无  
**输入**：`GameBoard(col=5, row=5)`  
**预期输出**：
- 棋盘为5x5矩阵，所有元素为None
- current_player = 0
- col = 5, row = 5, board_size = 25

#### TC-INIT-003: 边界尺寸初始化
**目的**：验证边界尺寸处理  
**前置条件**：无  
**输入**：`GameBoard(col=1, row=1)`  
**预期输出**：
- 棋盘为1x1矩阵
- 或抛出ValueError（根据实现决定）

---

### 4.2 落子操作测试 (TC-MOVE)

#### TC-MOVE-001: 有效落子
**目的**：验证有效落子更新棋盘  
**前置条件**：空棋盘  
**输入**：在位置(0,0)落子'X'  
**预期输出**：
- board[0][0] = 'X'
- 其他位置保持None

#### TC-MOVE-002: 重复落子
**目的**：验证不能在有棋子的位置落子  
**前置条件**：位置(0,0)已有'X'  
**输入**：在位置(0,0)落子'O'  
**预期输出**：
- 抛出IllegalMoveError或返回False
- 棋盘状态不变

#### TC-MOVE-003: 越界落子-负数
**目的**：验证负数坐标被拒绝  
**前置条件**：空棋盘  
**输入**：在位置(-1, 0)落子  
**预期输出**：抛出IndexError或ValueError

#### TC-MOVE-004: 越界落子-超出范围
**目的**：验证超出棋盘范围被拒绝  
**前置条件**：3x3棋盘  
**输入**：在位置(3, 3)落子  
**预期输出**：抛出IndexError或ValueError

#### TC-MOVE-005: 连续落子
**目的**：验证多次落子累积效果  
**前置条件**：空棋盘  
**输入**：
1. (0,0)落子'X'
2. (1,1)落子'O'
3. (2,2)落子'X'  
**预期输出**：
- board[0][0] = 'X'
- board[1][1] = 'O'
- board[2][2] = 'X'
- 其他位置为None

---

### 4.3 胜负判定测试 (TC-WIN)

#### TC-WIN-001: 行获胜-第一行
**目的**：验证第一行三连获胜  
**前置条件**：board = [['X','X','X'], [None,None,None], [None,None,None]]  
**输入**：调用胜负判定函数  
**预期输出**：返回('X', True)或winner='X'

#### TC-WIN-002: 行获胜-第二行
**目的**：验证第二行三连获胜  
**前置条件**：board = [[None,None,None], ['O','O','O'], [None,None,None]]  
**输入**：调用胜负判定函数  
**预期输出**：winner='O'

#### TC-WIN-003: 行获胜-第三行
**目的**：验证第三行三连获胜  
**前置条件**：board = [[None,None,None], [None,None,None], ['X','X','X']]  
**输入**：调用胜负判定函数  
**预期输出**：winner='X'

#### TC-WIN-004: 列获胜-第一列
**目的**：验证第一列三连获胜  
**前置条件**：board = [['X',None,None], ['X',None,None], ['X',None,None]]  
**输入**：调用胜负判定函数  
**预期输出**：winner='X'

#### TC-WIN-005: 列获胜-第二列
**目的**：验证第二列三连获胜  
**前置条件**：board = [[None,'O',None], [None,'O',None], [None,'O',None]]  
**输入**：调用胜负判定函数  
**预期输出**：winner='O'

#### TC-WIN-006: 列获胜-第三列
**目的**：验证第三列三连获胜  
**前置条件**：board = [[None,None,'X'], [None,None,'X'], [None,None,'X']]  
**输入**：调用胜负判定函数  
**预期输出**：winner='X'

#### TC-WIN-007: 对角线获胜-主对角线
**目的**：验证主对角线三连获胜  
**前置条件**：board = [['X',None,None], [None,'X',None], [None,None,'X']]  
**输入**：调用胜负判定函数  
**预期输出**：winner='X'

#### TC-WIN-008: 对角线获胜-副对角线
**目的**：验证副对角线三连获胜  
**前置条件**：board = [[None,None,'O'], [None,'O',None], ['O',None,None]]  
**输入**：调用胜负判定函数  
**预期输出**：winner='O'

#### TC-WIN-009: 无获胜-空棋盘
**目的**：验证空棋盘无获胜者  
**前置条件**：空棋盘  
**输入**：调用胜负判定函数  
**预期输出**：winner=None

#### TC-WIN-010: 无获胜-进行中
**目的**：验证游戏进行中无获胜者  
**前置条件**：board = [['X','O','X'], [None,'O',None], [None,None,None]]  
**输入**：调用胜负判定函数  
**预期输出**：winner=None

#### TC-WIN-011: 无获胜-满棋盘无连线
**目的**：验证满棋盘但无获胜者（罕见情况）  
**前置条件**：board = [['X','O','X'], ['X','O','O'], ['O','X','X']]  
**输入**：调用胜负判定函数  
**预期输出**：winner=None（实际是和棋）

### 4.4 和棋判定测试 (TC-DRAW)

#### TC-DRAW-001: 满棋盘和棋
**目的**：验证满棋盘无获胜者为和棋  
**前置条件**：board = [['X','O','X'], ['X','O','O'], ['O','X','X']]  
**输入**：调用和棋判定函数  
**预期输出**：is_draw = True

#### TC-DRAW-002: 未满棋盘非和棋
**目的**：验证未满棋盘不是和棋  
**前置条件**：board = [['X','O','X'], [None,None,None], [None,None,None]]  
**输入**：调用和棋判定函数  
**预期输出**：is_draw = False

#### TC-DRAW-003: 有获胜者非和棋
**目的**：验证有获胜者时不是和棋  
**前置条件**：board = [['X','X','X'], ['O','O',None], [None,None,None]]  
**输入**：调用和棋判定函数  
**预期输出**：is_draw = False

---

### 4.5 游戏状态测试 (TC-STATE)

#### TC-STATE-001: 游戏进行中
**目的**：验证游戏进行中状态  
**前置条件**：board = [['X',None,None], [None,'O',None], [None,None,None]]  
**输入**：调用状态检查函数  
**预期输出**：game_over = False, winner = None

#### TC-STATE-002: 游戏结束-获胜
**目的**：验证游戏结束状态（获胜）  
**前置条件**：board = [['X','X','X'], ['O','O',None], [None,None,None]]  
**输入**：调用状态检查函数  
**预期输出**：game_over = True, winner = 'X'

#### TC-STATE-003: 游戏结束-和棋
**目的**：验证游戏结束状态（和棋）  
**前置条件**：满棋盘无获胜者  
**输入**：调用状态检查函数  
**预期输出**：game_over = True, winner = None, is_draw = True

---

### 4.6 玩家切换测试 (TC-PLAYER)

#### TC-PLAYER-001: 首次切换
**目的**：验证首次落子后玩家切换  
**前置条件**：current_player = 0  
**输入**：执行一次落子  
**预期输出**：current_player = 1

#### TC-PLAYER-002: 多次切换
**目的**：验证多次落子后玩家正确切换  
**前置条件**：current_player = 0  
**输入**：执行3次落子  
**预期输出**：current_player = 1（奇数次切换）

---

### 4.7 游戏重置测试 (TC-RESET)

#### TC-RESET-001: 重置已进行游戏
**目的**：验证重置功能清除所有状态  
**前置条件**：棋盘有棋子，current_player = 1  
**输入**：调用reset_board()  
**预期输出**：
- 所有位置为None
- current_player = 0

#### TC-RESET-002: 重置已结束游戏
**目的**：验证重置已结束的游戏  
**前置条件**：游戏已结束（有获胜者）  
**输入**：调用reset_board()  
**预期输出**：
- 所有位置为None
- current_player = 0

## 6. 测试实现要求

### 6.1 代码规范
- 使用unittest框架
- 测试类命名：`Test<GameLogic>`
- 测试方法命名：`test_<场景>_<条件>`
- 添加中文文档字符串

### 6.2 断言使用
- 使用`assertEqual`验证相等
- 使用`assertTrue`/`assertFalse`验证布尔值
- 使用`assertIsNone`/`assertIsNotNone`验证None
- 使用`assertRaises`验证异常

### 6.3 测试隔离
- 每个测试用例独立运行
- 使用setUp/tearDown管理测试数据
- 避免测试间相互影响
